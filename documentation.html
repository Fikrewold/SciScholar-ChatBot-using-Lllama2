Proposed Solutions

Prototype Development

Technology Stack: Implemented in Python using Llama 2 (via CTransformers), which is an open-source LLM capable of understanding scholarly context.

Functionality: The chatbot handles a wide range of questions related to the publication. It extracts relevant content from local PDF data and provides contextually accurate answers.

Evaluation Approach

Business Metrics: Key KPIs include response accuracy, user satisfaction, and response time. This ensures the chatbot meets academic or enterprise needs.

Testing: The system is tested with diverse queries to ensure it accurately handles various research questions and maintains consistency across different datasets.

Assumptions

The chatbot assumes that input documents are well-structured academic papers and that queries align with the document's context.

It also assumes that the Llama 2 model can effectively generalize based on the provided data without requiring fine-tuning.

Reproducibility

To ensure reproducibility, the chatbot setup includes:

  • Predefined model versions and dependencies listed in requirements.txt.
  • Consistent dataset handling to maintain reliable response generation.
  • Clear documentation for re-running experiments and evaluating performance.

3. Installation

Clone the repository:

git clone https://github.com/Fikrewold/SciScholar-ChatBot-using-Lllama2.git

Navigate to the project folder:

cd SciScholar-ChatBot-using-Lllama2

Create a virtual environment (optional but recommended):

python -m venv venv
source venv/bin/activate  # Mac/Linux
venv\Scripts\activate     # Windows

Install dependencies:

pip install -r requirements.txt

Set up environment variables:

Copy or rename .env.example to .env, then fill in your keys (e.g., PINECONE_API_KEY, etc.) if you plan to use a vector DB.

Download or place the Llama 2 model file:

Ensure llama-2-7b-chat.ggmlv3.q8_0.bin (or your chosen variant) is in the root folder or update paths accordingly.

4. Usage

Run the main application:

python app2.py

This starts the Scholar-ChatBot. By default, it may run on localhost:8080 (or 127.0.0.1:8080) in debug mode.

Open your browser and visit:

http://localhost:8080

or the specified URL in your terminal output.

Ask questions or chat:

Use the web interface to query the chatbot. It will retrieve relevant documents (if integrated with Pinecone or a local vector store) and generate responses using the Llama 2 model via CTransformers.

5. Configuration & Customization

The main code for your chatbot logic resides in app2.py and the helper modules in src/helper.py and src/prompt.py. You can adjust:

  • Model path in app2.py if you rename or relocate the Llama 2 model file.
  • Embeddings or vector store in app2.py to use FAISS, Pinecone, or others.
  • Prompt templates in src/prompt.py to control how the model is prompted.

6. Troubleshooting

  • Model path issues:

    Ensure your model/llama-2-7b-chat.ggmlv3.q8_0.bin path is correct. If you see file not found errors, update the path in your code or rename the file to match.

  • Environment variables:

    If you’re using a vector DB like Pinecone, verify your .env file is loaded and contains valid keys. Check logs if connection fails.

  • Memory constraints:

    Llama 2 can be large. Ensure your system meets the RAM requirements for the chosen model. If you’re on limited hardware, use a smaller quantized model.
